## 游戏概述

### 基本信息
- **游戏名称**：《城市决策者》(City Governor AI)
- **游戏类型**：城市模拟 + 策略 + 决策管理 + AI对话
- **目标平台**：PC
- **目标受众**：城市规划爱好者、策略游戏玩家、政策研究者

### 核心创意
将《天际线2》的城市建筑系统与AI驱动的政府决策系统结合。玩家不仅要建设城市，更要通过任命副市长、设置部门权限、听取AI建议来做出政治决策。每个决策都影响城市的发展方向和市民满意度。

### 核心卖点
1. **动态政府系统**：自由定制部门结构
2. **AI角色扮演**：每个副市长有独立性格和立场
3. **真实政治博弈**：市议会投票、权力制衡、民意反馈
4. **决策追踪系统**：查看历史决策的长期影响
5. **高自由度**：新手自动平衡，高手可自定义一切

## 核心系统

### 1. 城市建筑系统

#### 1.1 基础元素
- **区域类型**：住宅、商业、工业、绿地、特殊建筑
- **基础设施**：道路、电力网、水系、污水处理
- **建筑生命周期**：建设 → 运营 → 升级/衰落 → 拆除

#### 1.2 资源系统
```
关键资源：
- 金钱（财政收入、支出）
- 电力（生产、消耗）
- 水资源（供应、污水）
- 人口（工作、居住、教育）
```

#### 1.3 指标系统
```
实时跟踪指标：
- 人口数量和密度
- 就业率和失业率
- 污染指数（空气、水、噪声）
- 拥堵率和平均通勤时间
- 教育指数和医疗指数
- 犯罪率
- 市民幸福度
- 城市财政状况
```

---

### 2. 政府部门结构

#### 2.1 基础部门（7个）

| 部门 | 职责 | 关键指标 |
|------|------|--------|
| **交通部** | 道路、公交、地铁规划 | 拥堵率、通勤时间 |
| **环保部** | 污染控制、绿化、垃圾处理 | 空气质量、绿化覆盖率 |
| **教育部** | 学校建设、教育投资 | 教育水平、覆盖率 |
| **卫生部** | 医院、医疗投资 | 健康指数、覆盖率 |
| **城建部** | 基础设施、电力水务 | 基建完善度 |
| **警察部** | 治安、警察局建设 | 犯罪率、安全感 |
| **财政部** | 预算分配、税收 | 财政收入、支出 |

#### 2.2 副市长系统
- **初始**：1-2名副市长
- **动态增加**：城市规模扩大时可聘请3-5名副市长
- **职责**：每位副市长管理2-3个相关部门

#### 2.3 副市长"性格"参数
```csharp
class ViceMayorPersonality {
    string name;              // 名字
    int age;                  // 年龄
    float developmentBias;    // 发展偏好 (0-1)
    float environmentBias;    // 环保偏好 (0-1)
    float welfareBalance;     // 福利偏好 (0-1)
    string[] communicationStyle;  // 沟通风格
    float[] proposalWeights;      // 建议权重
}
```

#### 2.4 市议会机制
- **投票触发**：预算调整>20%、新建大型项目、重大政策改变
- **投票成功条件**：市民满意度 > 60% 时投票通常通过
- **投票失败后果**：无法实施该决策，需要重新规划

### 3. AI决策建议系统

#### 3.1 建议触发
**主动建议**（AI主动提醒）：
- 拥堵率 > 80% → 交通部建议：扩建公交
- 污染 > 70% → 环保部建议：关闭污染工厂
- 失业率 > 15% → 教育部建议：开办职业培训

**定期报告**：
- 每个游戏月初，副市长汇报工作
- 展示部门表现数据
- 提出本月建议

#### 3.2 建议结构
```csharp
class DepartmentProposal {
    string departmentName;
    int priority;             // 1-5星
    string title;
    string problemAnalysis;
    ProposalOption[] options; // 多个应对方案
    float estimatedCost;
    string fundingSource;
    int expectedDuration;
    Dictionary<string, float> expectedImpacts;
}
```

#### 3.3 玩家回应方式
- **菜单选择**：完全同意、部分同意、延后、拒绝
- **自由对话**：输入自己的想法，AI回应
- **自定义方案**：玩家提出替代方案

#### 3.4 AI学习机制
```
系统记录：
- 玩家同意的建议类型（权重+30%）
- 玩家拒绝的建议类型（权重-20%）
- 玩家的决策时间（快速决策的偏好）
- 玩家的资金倾向（激进投资 vs 保守）

学习结果：
- AI调整建议频率和优先级
- AI调整建议内容和措辞
- AI预测玩家的下一步决策
```

### 4. 市民满意度系统

#### 4.1 满意度计算
```
总满意度 = (
    住房质量(30%) + 
    就业机会(20%) + 
    环境质量(15%) + 
    交通便利(15%) + 
    医疗教育(10%) + 
    安全感(10%)
) × 政治满意度系数

政治满意度系数 = 基于副市长支持度和政策认可
```

#### 4.2 失败条件
- ❌ **被罢免**：满意度 < 30% 连续3个月
- ❌ **破产**：负债 > 年收入200%
- ❌ **崩溃**：人口离开>50% 或 失业率>40%

---

### 5. 城市危机事件系统

#### 5.1 危机类型

| 危机 | 触发条件 | 立即影响 | 应对方案数 |
|------|--------|--------|--------|
| 污染爆发 | 污染>80% 6个月 | 健康↓20% | 3个 |
| 交通瘫痪 | 拥堵>90% | 失业↑10% | 3个 |
| 经济衰退 | 失业>25% 3个月 | 税收↓30% | 3个 |
| 犯罪浪潮 | 犯罪>70% | 满意度↓15% | 3个 |
| 自然灾害 | 随机 | 基建毁损 | 2个 |
| 疫情爆发 | 卫生不足 | 健康↓25% | 4个 |

#### 5.2 危机应对机制
- 每个危机有2-4个应对方案
- 每个方案有不同的成本、时间和效果
- 玩家的选择影响后续城市发展
- 可在新手模式关闭危机事件

### 6. 决策历史追踪

#### 6.1 决策记录
```csharp
class Decision {
    DateTime date;
    string title;
    string description;
    float investmentAmount;
    float expectedROI;
    Dictionary<string, float> actualResults;
    float satisfactionImpact;
    int monthsElapsed;

```

#### 6.2 长期影响评估
- 系统自动追踪每个决策的实际效果
- 对比预期与实际结果
- 生成"决策投资回报率"报告

## 游戏机制

### 游戏流程
1. **游戏开始**：分配初始资金和副市长
2. **城市规划**：建筑和规划城市
3. **接收建议**：AI副市长提出建议
4. **做出决策**：批准、拒绝或修改建议
5. **实施执行**：系统根据决策更新城市
6. **长期反馈**：观察决策的长期影响
7. **危机处理**：应对突发城市危机
8. **赢得/失败**：达成成就或触发失败条件

### 难度等级

| 模式 | 新手 | 普通 | 困难 | 自定义 |
|------|------|------|------|--------|
| 自动预算平衡 | ✅ | ❌ | ❌ | ⚙️ |
| 危机事件 | ❌ | ✅ | ✅频 | ⚙️ |
| 市议会投票 | 简单 | 中等 | 严格 | ⚙️ |
| 满意度衰减 | 慢 | 中 | 快 | ⚙️ |
