## 开发依赖关系

```
Phase 1 (框架) 
    ↓
Phase 2 (部门) + Phase 3 (AI) + Phase 4 (市议会)
    ↓
Phase 5 (危机) 
    ↓
Phase 6 (UI) + Phase 7 (集成测试)
```

**注**：Phase 2、3、4可以部分并行开发

---

## 版本控制策略

### 分支管理
- **main**：稳定发布版本
- **develop**：开发主分支
- **feature/phase-X**：各Phase特性分支
- **bugfix/XXX**：Bug修复分支

### 提交规范
```
[Phase-X] Feature description

Type: feat/fix/refactor/docs
Component: CitySystem/AISystem/UISystem等
```

## 测试策略

- **单元测试**：每个关键模块
- **集成测试**：系统间交互
- **功能测试**：玩家场景测试
- **性能测试**：帧率、内存、加载时间

## 风险评估

| 风险 | 概率 | 影响 | 缓解措施 |
|------|------|------|--------|
| AI对话复杂度 | 中 | 中 | 使用预定义回应库 |
| 性能问题 | 中 | 中 | 及早进行优化 |
| 美术资源不足 | 低 | 高 | 使用现成Asset包 |
| 超时交付 | 中 | 高 | 优先实现核心功能 |
